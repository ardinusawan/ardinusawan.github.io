Notes - Function score query
The function_score allows you to modify the score of documents that are retrieved by a query.
Docs:
Function score query | Elasticsearch Reference [7.9] | Elastic
The function_score allows you to modify the score of documents that are retrieved by a query. This can be useful if, for example, a score function is computationally expensive and it is sufficient to compute the score on a filtered set of documents.
https://www.elastic.co/guide/en/elasticsearch/reference/current/query-dsl-function-score-query.html#score-functions
function_score can be used with only one function like this:
Request
curl -X GET "localhost:9200/_search?pretty" -H 'Content-Type: application/json' -d' { "query": { "function_score": { "query": { "match_all": {} }, "boost": "5", "random_score": {}, "boost_mode": "multiply" } } } '
Shell
multi function
curl -X GET "localhost:9200/_search?pretty" -H 'Content-Type: application/json' -d' { "query": { "function_score": { "query": { "match_all": {} }, "boost": "5", "functions": [ { "filter": { "match": { "test": "bar" } }, "random_score": {}, "weight": 23 }, { "filter": { "match": { "test": "cat" } }, "weight": 42 } ], "max_boost": 42, "score_mode": "max", "boost_mode": "multiply", "min_score": 42 } } } '
Shell
Note: The scores produced by the filtering query of each function do not matter.
score_mode: the score of each function can be adjusted with a user defined weight
How to count score
Example
"functions": [ { "filter": { "match": { "test": "bar" } }, "random_score": {}, "weight": 23 },
Shell
weight * random score
if no key random_score, return weight
In case score_mode is set to avg the individual scores will be combined by a weighted average. For example, if two functions return score 1 and 2 and their respective weights are 3 and 4, then their scores will be combined as (13+24)/(3+4) and not (13+24)/2.
boost_mode
Search
Name
Desc
multiply
query score and function score is multiplied (default)
replace
only function score is used, the query score is ignored
sum
query score and function score are added
avg
average
max
max of query score and function score
min
min of query score and function score
min_score: To exclude documents that do not meet a certain score threshold the min_score parameter can be set to the desired score threshold.
max_boost: The new score can be restricted to not exceed a certain limit by setting the max_boost parameter. The default for max_boost is FLT_MAX.
