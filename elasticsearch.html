Sharding
Sharding is a way of dividing an index data volume into smaller parts which are called shards. This enables you to distribute data across multiple nodes within a cluster, meaning that you can store a terabyte of data even if you have no single node with that disk capacity. Sharding also increases performance in cases where shards are distributed on multiple nodes, because search queries can then be parallelized, which better utilizes the hardware resources that your nodes have available to them.

2.
P1 will send data to P0. Data will not inserted to R0 because it was slave, instead it will inserted to R1 P0 because P0 is master
3.

Retrieving
Flow:
1.
P1 get request, P1 will do routing to check where is the data
2.
P1 found data on R0. By default it will searching beside the node itself eventhough R0 is on same partition because it need to loads the work to other partition
3.
R0 return the data to P1

Request come to P1
2.

3.

4.
P0 update data to all replocation
